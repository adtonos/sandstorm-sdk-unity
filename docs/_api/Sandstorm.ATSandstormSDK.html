Class ATSandstormSDK

Shared object for VAST advertisements handling. This is the place where developer should start reading. ATSandstormSDK is an object containing a set of methods for generating and playing VAST advertisements. See Quick Guide for more information.

This method is also called automatically if an error occurred that prevents the further playback of ads. It does NOT remove the registered SandstormCallback. Must be called from coroutine or on main thread.

CreateBuilder()

Creates a new instance of SandstormVastUrlBuilder. Each time a new URL to VAST is needed it must be generated using the new builder. This method can only be called after the Start method has been executed and finished. To check the execution status of the start method you can use IsStarted. Builder is needed to enter the necessary additional data. One of the relevant elements is the Sandstorm key. Builder shall be passed to RequestForAds 

    var builder = ATSandstormSDK.CreateBuilder();
    builder.SetAdTonosKey(""); //THIS MUST BE SET TO VALUE FROM Sandstorm
    var requestResult = ATSandstormSDK.RequestForAds(builder: builder, view: CreateSandstormUnity());

The first necessary step in the project is to call the Initialize method. It should be called at the start of the application, when the other components are created. PauseAd()

Method that should be called whenever the application loses foreground (in the current implementation this is done via Dispatcher). Its task is to perform a pause on the currently playing material. If the method succeeds true value is returned and OnVastAdPaused method of SandstormCallback is called. If the pause fails while advertising material is available, the callback OnVastError (PauseAdError) from SandstormCallback is called. In the case where the pause is due to a change in the application foreground and does not affect the playback of the advertisement, only the value false will be returned.

RequestForAds(SandstormVastUrlBuilder, SandstormUnity)

The method call performs a query for the ads and then downloads them. When the process is complete and ready, the OnVastAdsLoaded method is called. From this moment on it is possible to call PlayAd. The method, depending on the current state, returns the appropriate value of the SandstormAdRequestResult enum informing what is currently happening. The object passed to the view parameter should always be a new instance because in case of a result other than Success the Dispose method will be called on it. All objects passed to this method after calling the Clear method will be destroyed and their instances should not be used again. In order to be able to play the next advertisement, the playback of the previous ones must be completed. See the example in the Quick Guide to see how the whole process can be combined.

Start(AdTonosConsent)

The library requires certain permissions and user consents for data collection. Therefore, one of the initial screens should show the user the terms and conditions containing the necessary information about the use and processing of personal data. This is done when the user first interacts with the application, so this is a good time to call the Start method. Additionally the Start method savesConsents. The method takes the AdTonosConsent consent flag as a parameter and then asks the user for the system permissions necessary for the application to work. This process is automated. The Start method must be called every time the application starts. Additionally, the LoadLatestConsents method can be used, which returns the most recently granted consents. Below is an example of how this can be done:

public void OnAppStarted() {
    if (CanShowPrivacyPolicyPopup()) {
        ShowPrivacyPolicyPopup(); //when ended shall call OnPrivacyPolicyPopupClosed
    } else {
        ATSandstormSDK.Start(ATSandstormSDK.LoadLatestConsents() ?? AdTonosConsent.None);
    }
}

public void OnPrivacyPolicyPopupClosed(AdTonosConsent consent) {
    ATSandstormSDK.Start(consent);
}

public bool CanShowPrivacyPolicyPopup()
{
    return ATSandstormSDK.LoadLatestConsents() == null;
}
